# M002 — 安装/登录/模型接入闭环

## Context

首用体验目前分散在安装、配置、模型、登录等多个路径中，需统一为低摩擦 onboarding 流程。

## Outcome

- 安装界面优化并简化流程（参考 PetClaw）。
- Codex GPT 自动接入，且支持自定义模型切换。
- 登录流程支持微信扫码、飞书扫码、支付宝扫码、手机号登录。

## Slices

### M002-S01 — 安装界面与流程简化

- [ ] T-M002-S01-001：重构安装界面结构与文案，突出进度和失败可恢复。
- [ ] T-M002-S01-002：压缩安装步骤，形成最短可用路径。
- [ ] T-M002-S01-003：补齐安装后检查与修复入口。

验收标准：
- 首次安装路径步骤减少且状态清晰。
- 失败后可继续或重试，不要求全量重来。

### M002-S02 — Codex GPT 自动接入与模型可替换

- [ ] T-M002-S02-001：默认创建 Codex GPT 可用配置。
- [ ] T-M002-S02-002：保留/增强手动配置入口用于切换自定义模型。
- [ ] T-M002-S02-003：新增模型接入状态提示与回退策略。

验收标准：
- 新用户可在最短路径内发起首条可用对话。
- 高级用户可替换模型，不破坏默认路径。

### M002-S03 — 多渠道登录中心

- [ ] T-M002-S03-001：设计统一登录态与账号绑定模型。
- [ ] T-M002-S03-002：实现微信、飞书、支付宝扫码登录入口。
- [ ] T-M002-S03-003：实现手机号登录与异常兜底流程。

验收标准：
- 四种登录方式入口可见且流程闭环。
- 登录失败提示可定位问题并可重试。

## Dependencies

- 需要第三方登录联调环境与应用凭据。

## Risks

- 第三方扫码协议和审核流程存在外部不确定性。
