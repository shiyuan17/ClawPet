# M006 — 发布稳定化

## Context

在完成核心重构后，需要统一进行质量与发布前验收，避免功能上线后回归。

## Outcome

- 动画与交互一致性达标。
- 关键链路通过回归与性能验证。
- 发布包可稳定安装、登录、对话与技能安装。

## Slices

### M006-S01 — 质量闸门与一致性验收

- [ ] T-M006-S01-001：回归测试（安装、登录、模型、聊天、技能、任务、报告）。
- [ ] T-M006-S01-002：动效一致性和无障碍可用性检查。
- [ ] T-M006-S01-003：关键性能指标验证（启动、切换、对话响应、快呼弹框）。

验收标准：
- 关键路径无 P0/P1 缺陷。
- 交互反馈和视觉一致性通过验收。

### M006-S02 — 发布准备与风险收口

- [ ] T-M006-S02-001：完善发布说明与升级路径。
- [ ] T-M006-S02-002：梳理已知风险与回滚预案。
- [ ] T-M006-S02-003：确认监控与问题上报机制。

验收标准：
- 发布流程可执行，回滚预案可演练。
- 已知风险有负责人和处理时限。

## Dependencies

- 依赖 M001-M005 完成。

## Risks

- 若前序里程碑缺少统一验收口径，M006 会变成临时补漏，导致发布窗口被动拉长。
