# M004 — 敏捷面板与可视化报告

## Context

当前运行数据分散在多个快照和日志中，缺少统一的任务执行视图和汇总报告。

## Outcome

- 上线敏捷面板，展示迭代目标、任务状态、阻塞项。
- 上线报告页面，完成数据去重汇总。
- 提供 agent 处理进度视图（当前任务、执行中、已完成）。

## Slices

### M004-S01 — 敏捷面板

- [ ] T-M004-S01-001：设计敏捷面板信息模型（任务、优先级、负责人、阻塞）。
- [ ] T-M004-S01-002：接入 staff/task 数据快照。
- [ ] T-M004-S01-003：增加筛选与状态切换能力。

验收标准：
- 面板可实时反映任务状态变化。
- 团队可快速识别阻塞与负责人。

### M004-S02 — 可视化报告与去重汇总

- [ ] T-M004-S02-001：构建报告聚合管道（日志、任务、员工、资源）。
- [ ] T-M004-S02-002：实现报告去重和统一时间线视图。
- [ ] T-M004-S02-003：增加当前任务/执行中 agent/完成任务汇总卡片。

验收标准：
- 报告信息无明显重复噪声。
- 可一屏查看“谁在做什么、做到哪一步”。

## Dependencies

- 依赖 M003 完成技能与任务侧数据标准化。

## Risks

- 若数据口径不统一，去重逻辑会失真，需先定义唯一键策略。
