# M001 — Chat 壳层重构

## Context

当前能力已覆盖聊天、员工、任务、技能、安装等模块，但前端主要承载在超大组件中。该里程碑目标是建立聊天优先壳层，先形成稳定骨架，再迁移功能。

## Outcome

- 主界面为微信式聊天布局。
- 左侧固定菜单包含：聊天、仪表盘、员工管理、员工招募、技能市场、任务管理。
- 形成可扩展壳层和路由容器。
- 完成前端超大组件拆分第一阶段。

## Slices

### M001-S01 — 信息架构与左侧菜单

- [x] T-M001-S01-001：冻结信息架构（菜单层级、命名、切换规则、默认落点）。
- [x] T-M001-S01-002：建立壳层布局（左菜单 + 主内容聊天承载区 + 顶部上下文栏）。
- [x] T-M001-S01-003：接入六大菜单入口并打通切换状态管理。

验收标准：
- 六个入口可稳定切换，状态可恢复。
- 主区域不发生布局抖动，窗口缩放行为一致。
