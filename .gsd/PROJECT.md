# 项目说明

## 这是什么

DragonClaw 是一个基于 Tauri 的桌面 AI 助手应用，将桌宠交互与 OpenClaw 控制台能力融合在同一 UI 中。

## 核心价值

用户可以在“聊天优先（chat-first）”的统一工作台中完成对话、配置、安装、员工协作与任务调度，无需频繁切换不同页面或手动编辑配置文件。

## 当前状态

应用已具备聊天、安装向导、技能市场、员工/任务快照、平台配置等基础能力，但前端承载集中在超大组件中，信息架构与交互路径仍偏“控制台堆叠”，需要重构为聊天界面承载式结构。

## 架构 / 关键模式

- 前端：Vue 3 + Pinia + Vite + TypeScript。
- 渲染与交互：PixiJS + GSAP 驱动桌宠动效与反馈。
- 桌面后端壳层：Tauri v2（Rust），通过命令处理 OpenClaw 运行时、配置读写与系统操作。
- 打包：构建阶段将 OpenClaw runtime / plugins 打包至 `src-tauri/resources`。

## 目标架构（本轮重构）

- 左侧固定功能菜单，主内容区采用聊天驱动承载（类似微信工作台）。
- “功能即面板”，聊天、仪表盘、员工、招募、技能、任务可在统一壳层切换。
- 安装、登录、模型接入走统一 onboarding 流程，减少首次使用认知负担。
- 将超大前端组件拆分为按领域模块，降低回归风险并提升迭代速度。
